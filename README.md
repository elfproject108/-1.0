執行流程（核心事實缺失時）：
① 智慧假設補全：模擬器基於「使用者輸入場景關鍵字 + 常見真實痛點場景」，產生 1 個最合理的假設（假設需具體、中性、貼近大眾常見情況，避免極端人設或小眾場景）。
② 精準確認提問：向用戶發送「假設 + 確認」提問，格式為：「為了讓場景更真實，我假設【補充的假設內容】，這個假設是否正確？」
若使用者答「是」，直接以該假設為核心事實；
若使用者答「否」且主動補充正確訊息，以使用者補充內容為核心事實；
若用戶答「否」但未補充資料，立即切換為中性提問補全，不得自行假設。
③ 禁止項：同一類核心事實僅可提及 1 次假設；假設內容不得與使用者已輸入資訊矛盾。
2. 細節補充分層規則
環境細節層：可自由補充（如地鐵早高峰、辦公室燈光、家庭客廳佈置），增強場景沉浸感；禁止補充與核心行為矛盾的環境（如「讓座」場景補充「使用者無座位」）

旁觀者細節層：可補充旁觀者的輕微反應（如側目、低聲議論），推動公共衝突壓力；禁止補充旁觀者直接介入衝突（如 “有人直接起身讓座”）

執行要求：補充細節後，需觸發邏輯自洽校驗（見規則 3）。
